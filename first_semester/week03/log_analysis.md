# 미션 컴퓨터 로그 분석 보고서

## 1. 개요

- **분석 대상 파일**: `mission_computer_main.log`
- **로그 기간**: 2023-08-27 10:00:00 ~ 2023-08-27 12:00:00
- **총 이벤트 수**: 35건

---

## 2. 로그 구조

로그는 CSV 형식으로 다음 세 필드로 구성된다.

| 필드 | 설명 |
|------|------|
| `timestamp` | 이벤트 발생 시각 (YYYY-MM-DD HH:MM:SS) |
| `event` | 이벤트 유형 (INFO / WARNING / ERROR 등) |
| `message` | 이벤트 상세 내용 |

## 4. 사고 원인 분석

### 문제 이벤트

```
2023-08-27 11:35:00,INFO,Oxygen tank unstable.
2023-08-27 11:40:00,INFO,Oxygen tank explosion.
```

### 분석

1. **미션 완료 후 발생**: 로켓 발사, 위성 배치, 착륙까지 모든 과정이 정상적으로 완료된 직후에 사고가 발생했다.
2. **산소 탱크 불안정 → 폭발**: 11:35에 불안정 징후가 감지됐지만 5분 후 폭발로 이어졌다. 사전 경보와 폭발 사이의 간격이 매우 짧아 대응이 이루어지지 못한 것으로 보인다.
3. **사전 이상 징후 없음**: 발사 전 점검 및 비행 전 과정에서 산소 탱크 관련 경고는 전혀 기록되지 않았다. 착륙 충격 또는 재진입 과정에서 누적된 물리적 손상이 원인일 가능성이 있다.
4. **시스템 전원 차단**: 폭발 20분 후 기지 전체 시스템이 전원 차단되어 이후 데이터는 기록되지 않았다.

### 결론

**산소 탱크의 갑작스러운 불안정으로 인한 폭발**이 사고의 직접적 원인이다.
발사·비행·착륙 단계에서는 모든 시스템이 정상이었으므로, 착륙 이후 지상 운용 단계에서의 탱크 결함 또는 충격 손상이 근본 원인으로 추정된다.
